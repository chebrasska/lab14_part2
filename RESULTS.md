## Ответы на вопросы

**1.** Во сколько раз параллельное перемножение матриц быстрее последовательного? Совпадает ли ускорение с количеством ядер?

> Ответ: Для матрицы 3x3 параллельная версия работает медленнее (0.53x). 
   Это связано с накладными расходами на создание процессов. Для маленьких 
   задач создание процессов занимает больше времени, чем само вычисление.

**2.** Как изменяется время выполнения при увеличении числа процессов в Pool?

> Ответ:С увеличением числа процессов время увеличивается из-за накладных 
   расходов на управление процессами. Для маленькой матрицы оптимально 
   использовать 1 процесс.

**3.** Почему для передачи результатов из процессов нельзя использовать обычные глобальные переменные?

> Ответ:Каждый процесс имеет свою копию памяти. Изменения в дочернем 
   процессе не видны в родительском.

**4.** Почему асинхронная версия в `01_sync_vs_async.py` выполняется быстрее синхронной, хотя использует только одно ядро?

> Ответ: Потому что во время ожидания (await asyncio.sleep()) управление 
   передаётся другим корутинам. Все три задачи выполняются "параллельно" 
   в одном потоке.


**5.** В каком порядке выводятся сообщения при запуске нескольких клиентов через `asyncio.gather()`?

> Ответ:Порядок недетерминированный. Сообщения выводятся по мере 
   завершения операций ввода-вывода.
